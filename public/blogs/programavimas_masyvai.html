Masyvai

Masyvai - visokiausių elementų sarašai, jie, skritingai nei aibės, gali turėti du, ar daugiau, vienodų daiktų. Ir daugelyje kalbų jie turi žinomą, pradinį dydį, kuris nesikeičia.
Beto, atmintyje jie neturį tarpų tarp jų dalių/daiktų/elementų, pavyzdžiui 4 raidžių masyvas atmintyje atrodys taip: 61 42 43 44 00, o ne: 61 00 00 00 00 00 00 00 42 43 00 00 00 44 00.


Čia mano masyvo pavyzdys parašytas naudojant kalbą: Java: 
String[] žodžiai = new String[5];
žodžiai[0] = "Pirmas elementas"; // Masyvų indeksai, gerose kalbose, prasideda nuo 0!
žodžiai[4] = "Paskutinis elementas"; // Ir taip... Čia 2 žodžiai, o ne 1 žodis...

String betkurisŽodis = žodžiai[2];
Čia sukuriu 5 teksto daiktų ilgio masyvą: {"", "", "", "", ""}. Tada pakeičiu kelius to masyvo elementus į savo tekstą, ir jis patampa: {"Pirmas elementas\0", "", "", "", "Paskutinis elementas\0"}. Galiausia pakeičiu berkurisŽodis į trečią žodžiai elementą ("")


Beveik tas pats masyvas su JavaScript: 
const žodžiai = []
žodžiai[0] = "Pirmas elementas"
žodžiai[4] = "Paskutinis elementas"

žodžiai.push("JavaScript leidžia padidinti masyvūs :(")

betkurisŽodis = žodžiai[2]
Javascript neklauso tipų, bet jei nėra = [] dalies, neišeis naudoti [0]/[4] ir .push().


Su pirmu C būdu: 
const char* žodžiai[5];
žodžiai[0] = "Pirmas elementas";
žodžiai[4] = "Paskutinis elementas";

const char* betkurisŽodis = žodžiai[2];
Šis būdas naudojamas greitams, žinomo dydžio masyvams, pavyzdžiui kubo sienoms, arba gyvūnų rušims.


Su antru C būdu: 
const char* *žodžiai = malloc(dabartinėSavaitėsDiena * 32); // C kalboje kintamuosius nežinomus kompiliacijos laiku reikia kurti taip.
// beto, aš, čia, kiekvienam žodžiui duodu 32 raides (baitus), nes dar nežinau kiek daugiausia raidžių man reikės, bet turbūt nedaugiau, kaip 32.
žodžiai[0] = "Pirmas elementas";
žodžiai[4] = "Paskutinis elementas";

const char* betkurisŽodis = žodžiai[2];
free(žodžiai);
Šis būdas naudojamas lėtesniams, bet kintamo dydžio masyvams.
