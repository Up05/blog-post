Kintamieji

Kompiuterių programavime kintamieji laiko duomenis, kaip skaičius, raides, kates, kačių sarašus ir daug daugiau. šios reikšmės/duomenys/daiktai yra, plius, minus tiesiog „nuliukai" ir „vienetukai", pavyzdžiui skaičius 32 būtų 0010 0000, raidė A būtų 0100 0001. Nuliai kairėje nereikšmingi, kaip ir „aš turiu 000000000000000000000000001 saldainį".

Iki 11 klasės Matematikoje mes dažnai ieškome kintamųjų reikšmių, arba jų reikšmių intervalų, pavyzdžiui 3x + 9 = 0; -3x = 9; (3x = 9) / -3; x = -3; Bet be to Matematikoje, ir ypač Fizikoje mes turime formules, kur mes naudojama turbūt panašiausius į I.T. kintamuosius, pavyzdžiui diskriminantas. D = b² - 4ac; ir suskaičiavę D mes naudojame tai ieškodami x₁ ir x₂, x₁ = (-b - kvadratinė_šaknis_iš(D)) / 2a; x₂ = (-b + kvadratinė_šaknis_iš(D)) / 2a; Be to, Matematikoje ir Fizikoje x = y reiškia, kad x YRA TAS PATS, KAIP y; Informatikoje: x = y reiškia: x DABAR BUS y, mes turime == patikrinti lygybę.

Skirtingos kalbos dažniausia turi truputi skirtingus kintamųjų skelbimo būdus, bet visos geros(kurias aš naudoju) kalbos naudoja formulę panašę į šią: apibūdinimas/savybės tipas pavadinimas [ = reikšmė].

// C / C++ / Java
int geraSkaičiavimoSistema = 17;

C, C++ ir Java pačiame priekyje gali turėti kintamųjų apibūdinimus, kaip extern(iš kažkur kitur), final(nesikeis/konstanta) ir daugiau.

// Javascript
let geraSkaičiavimoSistema = 17;

Javascript neturi tipų (arba tipų, kuriuos programuotojai galėtų pasakyti). taigi čia yra tik const, konstanta, let ir var, kintamasis ir apibrėžimai, kaip static ir viskas! Beje: const PI = 3.14, o ne const let PI = 3.14.

// Rust
let gera_skaičiavimo_sistema:u8 = 17; // galima būtu vadinti ir geraSkaičiavimoSistema, bet kalbos kūrėjai pasakė, kad būtų gerai naudoti snake_case kintamųjų pavadinimams.

Rust turi tipus, kaip C/C++, bet čia, kaip aišku bus skaičius, taigi pasakyti nebūtina. Aš pasakau, kad man duotu mažesnio dydžio skaičiaus tipą, u8 reiškia unsigned int su aštuoniais bitais. unsigned reiškia be ženklo(šiuo atveju(ir visais kitais...), be neigiamų skaičių), int reiškia sveikasis skaičius (be kablelio), 8 bitai, tai 0000 0000, nes int dažniausia turi: 00000000 00000000 00000000 00000000.



Kintamųjų tipai

Kintamieji, kaip jau sakiau, turi tipus, jie šiaip nieko nereiškia, tik kiek atminties paimti programai ir į kurias funkcijas juos įdėti. Čia yra 4 primityvūs tipai, jie rašomi mažąja pirma raide ir du Daiktai/Objektai/... jie didžiąja. Primityvūs ir nurodomieji tipai turi kelius skirtumus, apie kuriuos, aš, tikiuosiu, kad pakalbėsiu ateityje.

Mano pavadinimas C C++ Java Rust Trumpas aprašimas
Sveikasis skaičius int int int isize Skaičius be kablelio (Integer/Sveikasis skaičius). Dažniausia turi 32 bitus, arba 4 baitus.
Taip, arba ne - bool boolean bool Loginė reikšmė (būlis), taip arba ne, beje pačioje C kalboje nėra šio tipo, bet ji turi: <stdbool.h>, kas turi: bool. Dažniausia iš 1 baito.
Trupmenos float float float fsize Skaičius su kableliu, float(floating point number) nes kablelis/taškas keičia vietą. Dažniausia sudarytas iš 32 bitų, arba 4 baitų.
Viena raidė char char char char Viena raidė, jei naudoja ASCII, tai 1 baitas, jei ne: dažniausia turi "w" prieš "char" ir naudoja iki 4 baitų.
Tekstas* char[] std::string String String Tekstas, char[] negalima padidinti raidžių skaičiaus, reikia sukurti naują char masyvą, o kitus galima. String ateina iš "surištų" raidžių.
Objektas/Daiktas struct Daiktas Daiktas Object/Daiktas Daiktas Objektus, arba Daiktus aš, turbūt, paaiškinsiu vėliau, nes tai nevisai mažas dalykas, bet jų tipai rašomi Didžiaja pirma raide.
* Tekstą aš padėjau prie kintamųjų, nes jis labai dažnai naudojamas, bet tai dažniausia, arba paprastas Objektas, arba kintamųjų masyvas/sąrašas.

Ir viskas, plius, minus

Kintamieji dar gali laikyti: funkcijų vietos ir kitų daiktų vietų rodyklius(pointer), masyvūs, ir daug daugiau. Bet šie daiktai yra nevisai pradmenyse, taigi, gal vėliau, o gal ir -- ne, dar nežinau.
